# 管理 claude.md 的项目记忆文件

CLAUDE.md 是放在项目根目录的一个 Markdown 文件，Claude Code 在每次启动时会自动读取它作为上下文。你可以把它理解为「给 AI 的备忘录」，里面记录着：

- 项目的构建、测试、部署命令
- 目录结构和模块关系
- 代码风格约定
- 各种非显而易见的坑和 workaround
- 环境配置的特殊之处


# 使用方法
claude-md-management 插件包含两个 Skill：

1. `/claude-md-management:revise-claude-md`
2. `/claude-md-management:claude-md-improver`

它们解决的是同一个大问题 —— **让 CLAUDE.md 保持有用且准确**，但切入的角度完全不同。


## /revise-claude-md
会话结束时的「学习笔记」

### 它解决什么问题？

你刚跟 Claude Code 完成了一轮高效的协作，过程中发现了一些新的项目知识：也许是一个隐藏的环境变量要求，也许是某个测试命令需要特殊参数，也许是踩了一个框架的坑。这些都是宝贵的上下文，但如果不记下来，下次开新会话就全忘了。

`/revise-claude-md` 就是在这个时刻使用的 —— 它会回顾当前会话中发生了什么，把有价值的发现提炼成简洁的条目，添加到 CLAUDE.md 中。

### 它的工作流程

1. **回顾反思**：分析本次会话中缺失了哪些上下文，哪些信息如果一开始就有就能更高效
2. **定位文件**：找到项目中相关的 CLAUDE.md 文件
3. **草拟新增条目**：每条信息精炼为一行，格式为 `<命令或模式>` – `<简短描述>`
4. **展示差异**：在应用前先展示修改的 diff
5. **经你确认后写入**

### 什么时候用它？

- 刚完成了一次有收获的 debug 会话
- 发现了项目里的隐藏规则或约定
- 找到了某些命令的正确使用方式
- 踩了一个坑，希望下次 Claude 能直接避开

简单说：**每次会话结束前，觉得学到了什么，就跑一下 `/revise-claude-md`**。

##  /claude-md-improver

全面体检与质量审计

### 它解决什么问题？

你的 CLAUDE.md 可能已经写了一段时间了。项目结构变了、依赖升级了、某些命令已经废弃了，但 CLAUDE.md 还停留在老版本。或者你刚接手一个项目，里面的 CLAUDE.md 写得很粗糙，缺少关键信息。

`/claude-md-improver` 会对你所有的 CLAUDE.md 文件做一次全面审计，打分评级，然后给出具体的改进建议。

### 它的工作流程

1. **发现阶段**：扫描项目中所有 CLAUDE.md 文件（包括根目录的 `CLAUDE.md`、个人覆盖文件 `.claude.local.md`、全局文件 `~/.claude/CLAUDE.md`、以及 monorepo 中各子包的文件）
2. **质量评估**：按照详细的评分标准进行打分
3. **生成质量报告**：输出评级（A=90-100, B=70-89, C=50-69, D=30-49, F=0-29）
4. **提出改进方案**：给出具体可执行的修改建议
5. **经你确认后执行更新**


### 什么时候用它？

- 项目经历了一轮大重构或依赖升级后
- 新接手一个项目，想评估 CLAUDE.md 的质量
- 定期维护（比如每月做一次审计）
- 觉得 Claude Code 最近的表现不够好，怀疑是 CLAUDE.md 过时了

简单说：**定期体检用 `/claude-md-improver`**。


## 核心区别一览

||/revise-claude-md|/claude-md-improver|
|---|---|---|
|**定位**|会话级别的增量更新|项目级别的全面审计|
|**触发时机**|一次有收获的会话结束时|定期维护或大变更后|
|**工作范围**|只关注本次会话中发现的新知识|扫描全部 CLAUDE.md 文件并与代码库对照|
|**输出风格**|轻量，几行精炼的新增条目|详细的质量报告 + 评分 + 改进建议|
|**类比**|随手记笔记|年度体检|
|**使用频率**|频繁（每次有收获的会话后）|偶尔（大变更或定期维护时）|

## 实际使用建议

### 日常工作流

建议将这两个 Skill 融入你的日常开发节奏：

1. **每次有意义的会话结束前**，运行 `/claude-md-management:revise-claude-md`，把新发现的知识沉淀下来
2. **每周或每次大改动后**，运行 `/claude-md-management:claude-md-improver`，做一次全面检查

### CLAUDE.md 的内容原则

不管用哪个 Skill，好的 CLAUDE.md 内容都遵循这些原则：

**应该写：**

- 构建、测试、lint、部署的具体命令
- 模块之间的依赖关系和架构图
- 踩过的坑和对应的 workaround
- 有效的测试策略
- 配置文件中的关键设置

**不应该写：**

- 看代码就能明白的显而易见的信息
- 通用的编程最佳实践
- 一次性的修复，不太可能再次遇到
- 一句话能说完的事用了一段话来描述

### CLAUDE.md 文件的层级

值得注意的是，Claude Code 支持多层级的 CLAUDE.md 文件：

- `~/.claude/CLAUDE.md` —— 全局默认配置，适用于你所有的项目
- `./CLAUDE.md` —— 项目级配置，会随代码提交，团队共享
- `./.claude.local.md` —— 个人本地覆盖配置，不会提交到版本库
- 子目录中的 CLAUDE.md —— 适用于 monorepo 中各子包的特定配置

`/claude-md-improver` 会扫描并审计所有这些层级的文件，确保每一层都发挥应有的作用。

## 总结

CLAUDE.md 是 Claude Code 理解你项目的关键入口。维护得好，Claude 就像一个熟悉项目的老同事；维护得差，Claude 就只是一个什么都不知道的新手。

**claude-md-management** 插件用两个 Skill 覆盖了维护 CLAUDE.md 的两个核心场景：

- `/revise-claude-md` = **随时记录**，把每次会话的收获沉淀为项目记忆
- `/claude-md-improver` = **定期审计**，确保项目记忆与代码库同步、质量达标

把这两个工具用起来，你的 Claude Code 体验会有质的飞跃。
